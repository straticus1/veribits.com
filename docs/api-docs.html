Overview

The VeriBits API provides programmatic access to our trust verification platform. You can verify file hashes, email addresses, and blockchain transactions with enterprise-grade security and real-time threat intelligence.

Base URL

https://api.veribits.com/api/v1

Content Type

All requests must use Content-Type: application/json

Response Format

All responses return JSON with a consistent structure:

{
  "success": true,
  "message": "Success",
  "data": { ... },
  "timestamp": "2024-01-15T10:30:00Z"
}

Authentication

VeriBits API uses JWT (JSON Web Tokens) for authentication. Include your token in the Authorization header:

Authorization: Bearer your_jwt_token_here

Webhooks

Webhooks allow you to receive real-time notifications about verification events.

POST/webhooks

Register a new webhook endpoint

ðŸ”’ Requires authentication

Request Body

{
  "url": "https://your-app.com/webhooks/veribits",
  "events": ["verification.completed", "quota.warning"]
}

Available Events

  • * - All events
  • verification.completed - When a verification is completed
  • quota.warning - When quota usage reaches 80%
  • quota.exceeded - When quota is fully used

Webhook Signature Verification

All webhook payloads are signed with HMAC-SHA256. Verify the signature using the X-VeriBits-Signature header:

X-VeriBits-Signature: sha256=calculated_signature

Rate Limits

API endpoints are subject to rate limiting to ensure fair usage:

Endpoint Type Rate Limit Window
Authentication 30 requests 1 minute
Verification Monthly quota based on plan Per month
General API 100 requests 1 minute

Rate Limit Headers

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1642248000

Error Handling

The API uses conventional HTTP response codes and returns error details in JSON format:

Status Code Meaning
200 OK - Request successful
400 Bad Request - Invalid request data
401 Unauthorized - Invalid or missing authentication
403 Forbidden - Insufficient permissions
404 Not Found - Resource doesn't exist
422 Unprocessable Entity - Validation errors
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error - Server error

Error Response Format

{
  "success": false,
  "error": {
    "message": "Validation failed",
    "code": 422,
    "validation_errors": {
      "email": ["The email field is required"]
    }
  },
  "timestamp": "2024-01-15T10:30:00Z"
}

Common Error Codes

  • INVALID_TOKEN - JWT token is invalid or expired
  • QUOTA_EXCEEDED - Monthly verification quota exceeded
  • RATE_LIMIT_EXCEEDED - Too many requests in time window
  • VALIDATION_FAILED - Request data validation failed
  • RESOURCE_NOT_FOUND - Requested resource doesn't exist
